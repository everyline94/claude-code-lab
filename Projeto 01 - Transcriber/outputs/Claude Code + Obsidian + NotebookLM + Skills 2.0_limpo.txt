The combination of Claude Code, NotebookLM, Obsidian, and the Skill Creator creates one of the most powerful research workflows available. When these tools are combined in a practical, simple-to-set-up workflow, Claude Code becomes an absolute research monster. The key insight is that this isn't about any one specific use case — the value lies in adapting this template to whatever source of information you need, whether that's PDFs, articles, text, or any other data source. The workflow is highly flexible and can adapt to individual needs.

The workflow operates as follows: inside Claude Code, research is conducted via a data source (in this case YouTube). A specific skill handles the search and retrieval. That data is then sent to NotebookLM via Claude Code, where NotebookLM performs analysis and generates deliverables — podcasts, videos, infographics, slide decks, or other formats. Everything is passed back to Claude Code, and the entire process is executed through skills.

The individual subskills can be combined into a single "super skill" using the Skill Creator. Instead of telling Claude Code step by step which skill to run, one unified skill handles the entire pipeline from search to analysis to deliverable generation.

Obsidian supercharges this workflow by serving as the persistent knowledge layer. All the data analyzed, the preferred analysis methods, deliverable formats, and thinking patterns are recorded by Claude Code as markdown files within an Obsidian vault. For the human user, Obsidian provides visual insight into the files, their connections, and graph-based relationships. For Claude Code, those markdown files are transparent and easily accessible, making it simpler to find what it needs.

Over time, the workflow becomes a self-improving loop. The claude.md file acts as a "brain within a brain" — while the vault serves as a second brain containing ideas and research, the claude.md file tells Claude Code what it all means: conventions for communication, deliverable preferences, and work style. The more the workflow runs, the more data is recorded, and Claude Code continues to build a corpus of knowledge about how the user likes to work. This creates a symbiotic relationship where all the tools reinforce each other.

To set this up, skills need to be created first. The Skill Creator plugin is installed via the slash plugin command. Once installed, describing the desired skill in natural language is enough — for example, creating a skill that searches YouTube and returns structured video results using yt-dlp. The Skill Creator automatically generates the skill inside the .claude folder.

For the NotebookLM integration, since NotebookLM doesn't have a public-facing API, the connection uses the notebook-lm-pi GitHub repository. Installation involves running the provided commands in the terminal, then authenticating with the notebook-lm login command, which opens a browser window for sign-in. To teach Claude Code how to use NotebookLM, a skill is created either through the repo's built-in skill install command or by feeding the repo to the Skill Creator. The NotebookLM skill enables creating notebooks, adding up to 50 sources (from Drive, text files, YouTube, etc.), and generating deliverables like audio reviews, mind maps, flashcards, and infographics.

Once the individual skills exist, the Skill Creator combines them into a single pipeline skill. A stream-of-consciousness description of the desired workflow is sufficient — specify the YouTube search, the NotebookLM analysis, and the deliverable requirements. The Skill Creator handles the rest.

In practice, the workflow executes everything in one command. The pipeline skill searches for videos, sends them to NotebookLM for analysis, and returns both the research and requested deliverables. All AI processing for analysis is handled by NotebookLM (offloaded to Google), so no additional tokens are consumed by Claude Code. Text analysis is typically fast, while deliverables like infographics take a few minutes and full slide decks can take up to 15 minutes.

The research output is saved as a markdown file inside Obsidian, complete with backlinks to related documents and visible in the knowledge graph. Over time, the aggregate of these markdown files shows Claude Code how the user works. Updating the claude.md file to reflect the latest work style, analysis preferences, and output conventions is as simple as asking Claude Code to do so. This refinement compounds — while a week of use shows modest improvement, months and years of accumulated documents and conversations produce a significant and lasting effect.

The core principle is that any workflow can be plugged into this template. The specific data source is interchangeable, and even NotebookLM can be swapped out. What matters is maintaining the structure: a workflow, Obsidian for persistence and refinement, and skills via the Skill Creator. This combination puts a powerful, self-improving research system at your fingertips.
